# Release 1.0.0

## Release Type
- [x] Major (X.0.0) — Breaking changes
- [ ] Minor (0.X.0) — New features, backward compatible
- [ ] Patch (0.0.X) — Bug fixes

## Semantic Version Recommendation
- Recommended version: `1.0.0` from current package baseline `0.1.0`.
- Rationale: ADR-001 approved breaking user-facing changes: removal of `autoRetry`, removal of raw string `ssoLoginCommand`, and npm entrypoint migration from source TypeScript to compiled `dist/` server plugin output.
- Note: `package.json` version remains `0.1.0` until final release sign-off; no version bump, tag, publish, push, commit, or PR was performed in this release-management pass.

## Changelog

### Added
- Dist publishing model: package metadata now targets `dist/index.js` and `dist/index.d.ts`, exposes `./server`, publishes only `dist/`, and declares `oc-plugin: ["server"]`.
- Build and release scripts: `build`, `prepack`, `test`, and `typecheck` are available for release validation.
- Package contract tests for OpenCode server plugin metadata and build script expectations.

### Fixed
- OpenCode API compatibility: plugin now compiles against `@opencode-ai/plugin` `^1.14.20` and uses the official `tool.execute.after(input, output)` hook shape.
- Auth error detection now reads the official hook result from `output.output` while preserving centralized `AWS_AUTH_ERROR_PATTERNS`.
- Refresh behavior is covered by tests for successful execution, failed refresh logging, max retry limiting, concurrent refresh coalescing, ignored tools, and ignored non-AWS output.

### Changed
- Custom SSO refresh command configuration now uses structured data: `{ "command": string, "args"?: string[] }`.
- Diagnostics are explicitly documented as `client.app.log` troubleshooting logs rather than clickable or session-visible UI notifications.
- Release validation now requires `bun run build && bun run typecheck && bun test`.

### Removed
- Removed unsupported `autoRetry`; users must rerun the failed AWS command after credentials refresh if needed.
- Removed raw shell-string `ssoLoginCommand` support; migrate to structured command and args.
- Removed stale local OpenCode hook type shim `src/types.d.ts` and stale retry-state assumptions.

## Release Notes (external)
This release modernizes `opencode-plugin-aws-auth-refresh` for current OpenCode server plugin packaging and APIs. The package now publishes compiled `dist/` output, declares server plugin metadata, and compiles against `@opencode-ai/plugin` `^1.14.20`. AWS auth detection and refresh behavior remain focused on `bash` and `task` tool output, with diagnostics logged through `client.app.log`.

Breaking migration notes: remove `autoRetry` from configuration, rerun failed AWS commands manually after refresh if needed, and replace raw `ssoLoginCommand` strings with structured `{ "command": "aws", "args": [...] }` configuration.
